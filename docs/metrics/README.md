## How this works

A nightly GitHub Action calls the public GitHub Releases API, sums the `download_count` reported per asset, and writes a snapshot to `snapshots/YYYY-MM-DD.json`. `latest.json` always points at the newest snapshot. Schema documented in `SCHEMA.md`.

Limitations:

- Counts are downloads, not installations — an asset can be downloaded and never installed.
- Diffing snapshots can yield negative deltas if a release is renamed or an asset is removed; treat the cumulative number as authoritative, not daily deltas.
- Future install sources (winget, Homebrew, Flathub, Snap) will land as additional `source` files using the same schema.
